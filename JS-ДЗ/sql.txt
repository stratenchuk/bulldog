
1. Todo-лист
Визуальная диаграмма базы данных

+----------------+       +-----------------+
|     Users      |       |     Tasks       |
+----------------+       +-----------------+
| user_id (PK)   |<----+ | task_id (PK)    |
| username        |     | | user_id (FK)    |
| email           |     | | title           |
| password_hash   |     | | description      |
| dat             |     | | due_date        |
+----------------+       | | status          |
                         | | created_at      |
                         | | updated_at      |
                         | +-----------------+
                         |
                         |  +-----------------+
                         |  |   tasks_1     |
                         |  +-----------------+
                         +--| id (PK) |
                            | name              |
                            +-------------------+
Описание таблиц
Users

user_id (PK): Уникальный идентификатор пользователя.
username: Имя пользователя.
email: Электронная почта пользователя.
password_hash: Хэш пароля для аутентификации.
dat: Дата и время создания учетной записи.

Tasks

task_id (PK): Уникальный идентификатор задачи.
user_id (FK): Внешний ключ, ссылающийся на user_id в таблице Users.
title: Заголовок задачи.
description: Описание задачи.
due_date: Дата выполнения задачи.
status: Статус задачи (например, "выполнено", "в процессе", "отменено").
created_at: Дата и время создания задачи.
updated_at: Дата и время последнего обновления задачи.
Tasks_1

id (PK): Уникальный идентификатор категории.
name: Название категории.
Связи между таблицами
Users и Tasks: Один ко многим (1:N). Один пользователь может иметь много задач, но каждая задача принадлежит только одному пользователю.

2. Интернет-магазин

+----------------+       +----------------+       +----------------+
|    Customers    |       |     Orders     |       |   Products     |
+----------------+       +----------------+       +----------------+
| customer_id (PK)|<----+| order_id (PK)  |       | product_id (PK)|
| name            |      | customer_id (FK)|-----> | name           |
| email           |      | order_date      |       | description    |
| phone           |      | total_amount    |       | price          |
+----------------+      +----------------+       +----------------+
                             |
                             |
                             |
                             v
                     +----------------+
                     | Order_Items    |
                     +----------------+
                     | order_item_id (PK)|
                     | order_id (FK)      |
                     | product_id (FK)    |
                     | quantity            |
                     | price               |
                     +----------------+
Описание таблиц и связей
Customers:

customer_id (PK): Уникальный идентификатор клиента.
name: Имя клиента.
email: Электронная почта клиента.
phone: Телефон клиента.
Orders:

order_id (PK): Уникальный идентификатор заказа.
customer_id (FK): Внешний ключ, ссылающийся на customer_id в таблице Customers.
order_date: Дата заказа.
total_amount: Общая сумма заказа.
Products:

product_id (PK): Уникальный идентификатор продукта.
name: Название продукта.
description: Описание продукта.
price: Цена продукта.
Order_Items:

order_item_id (PK): Уникальный идентификатор элемента заказа.
order_id (FK): Внешний ключ, ссылающийся на order_id в таблице Orders.
product_id (FK): Внешний ключ, ссылающийся на product_id в таблице Products.
quantity: Количество продукта в заказе.
price: Цена продукта на момент заказа.

Связи между таблицами
Customers и Orders: Один ко многим (один клиент может иметь много заказов).
Orders и Order_Items: Один ко многим (один заказ может содержать много элементов).
Products и Order_Items: Один ко многим (один продукт может быть в многих заказах).
Эта структура позволяет эффективно управлять данными интернет-магазина, обеспечивая связь между клиентами, заказами и продуктами.
